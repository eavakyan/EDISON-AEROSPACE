Executive Summary\n\nThe Edison Aerospace corporate website is a **fully static site** with **no runtime database**. All content is managed as structured data at **build time** using Astro's Content Collections API backed by **JSON/YAML/Markdown files** validated with **Zod schemas**. This document defines the canonical data model, relationships, validation rules, and content file structures that Astro compiles into static HTML at build time.\n\nThe \"database\" for this project is the filesystem under `src/content/`, with Zod schemas serving as the schema definition layer, and Astro's Content Collections serving as the query layer.\n\n---\n\n## 2.
